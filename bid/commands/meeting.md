# /bid:meeting — 会议节点一键流程

**$ARGUMENTS**

## 模式分派

1. **激活 skill** `bid-playbook`(目录规范/纪要编年/口径红线是本命令的基准约束)。
2. 参数含 `--prep` → 走**会前模式**;否则走**会后模式**(默认)。
3. **定位会议**:参数是纪要文件路径 → 直接用;是日期 → 在 `meeting/` 按 `YYYY-MM-DD-*` 匹配;无参数 → 从当前会话上下文推断最近讨论的会议。推不出 → 停下问用户,不猜。

## 会后模式(默认):归档 + 口径变更提取

1. **纪要归档打标**:落 `meeting/YYYY-MM-DD-主题.md`,全项目共享编年(多主体项目也**不**按项目切分 meeting 目录——同一批人常跨项目开会,切分会造成割裂);每篇文首打「涉及项目」标签。**停止条件**:目标文件已存在 → 只展示 diff 预览,由用户决定覆盖与否,绝不静默覆盖。
2. **口径变更提取**:通读纪要,逐条列「会议定案表」:旧口径 → 新口径 → 影响的文档类别(客户向/内部/生成器)。判据:数字、措辞、范围、定位四类变化都算;权威定义与决策过程留痕于纪要本身,正式交付物后续只写自解释的事实陈述。无变更也要显式记一行「本次会议无口径变更」。
3. **级联分流**:凡变更触及已锁定数字/措辞 → **本命令不直接改交付物**,列出爆炸半径预览(生成器源/派生文档/memory 条目),提示用户走 `/bid:sync`(加载 skill `single-source-sync`:改源 → 重生成 → 级联 → grep 残留)。判据:每条变更都标注了「无需级联 / 需走 sync」。
4. **memory 固定末步**:把每条口径/定位/呈现决策即时写入项目 memory(索引行 + 详情条目),**明确记录被废弃的旧口径**与保留的分层例外(如「客户向已撤、内部故意保留」),防未来会话把已撤销的说辞重新引入。判据:会议定案表每一行在 memory 有对应条目。
5. **提交预览**:按功能分组(纪要/memory)列出显式路径的提交预览,点名列出不碰的无关预存改动;**只预览,不自动 commit**。

## 会前模式(--prep):准备包五件套

1. **激活 skill** `presales-tactics`(体量摸底/分档锚定/谈判红线的方法论来源),输入 = 会议主题 + 现有材料(客户向 docs/、内部底稿、memory 锁定口径表)。无额外输入时从现有材料自给自足生成,未知客户事实压成摸底问题清单带去现场,准备不停。
2. **生成五件套**,全部落内部目录(绝不写入客户向 docs/,绝不外发):
   - **讲解脚本**:按交付物阅读顺序组织的口播主线;
   - **关键数字速查卡**:只收锁定口径数字。判据:卡上每个数字能在客户向交付物中找到同值,内部区间/中位数一个不进卡;
   - **多视角模拟 Q&A**:至少老板/财务/风控三视角,每条附「别说」陷阱提示;
   - **『别说』红线清单**:内部数字勿口播勿投屏、未锁定承诺不出口等,先成文再上桌;
   - **口径桥**:枚举客户可能抓到的跨文档矛盾点(如工期表述、金额算例、合规档位这类多文档各自演化易错位的维度)+ 每点应对话术。
3. **对抗审校**:加载 skill `adversarial-review`,对速查卡与口径桥做数字一致性核对(与客户向交付物逐值 grep 比对)+ 红线泄漏检查。判据:零不一致才算通过;发现不一致 → 记入口径桥而非顺手改交付物。
4. **演示预案**:会上要演示网页/原型时,检查 CDN 字体/图标等网络依赖,写明断网降级表现与现场网络方案(手机热点),列入会前逐项检查清单并提前实测。
5. **memory 固定末步**:生成过程中发现的跨文档矛盾点、口径澄清、摸底问题清单落 memory,防下次会前重挖。
6. **提交预览**:同会后模式第 5 步,只预览不 commit。

## 常用用法

| 场景 | 命令 |
|---|---|
| 会后:归档指定纪要文件 | `/bid:meeting meeting/2026-01-15-需求澄清.md` |
| 会后:按日期定位当日纪要 | `/bid:meeting 2026-01-15` |
| 会后:刚在会话里聊完的会 | `/bid:meeting` |
| 会前:为下周会议出准备包 | `/bid:meeting 2026-01-20 --prep` |

## 安全保证

- **只预览不执行破坏性动作**:commit、覆盖已有文件一律先出预览,等用户确认。
- **不改交付物**:口径变更的落地统一走 `/bid:sync`,本命令只提取、登记、分流。
- **分层纪律**:准备包属内部材料,物理落内部目录;速查卡/讲解脚本中出现任何非锁定口径数字视为红线泄漏,审校不通过。
- memory 写入是唯一默认执行的落盘动作(追加式,不覆盖旧条目)。
