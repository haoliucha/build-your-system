# /bid:sync — 口径变更级联同步

**$ARGUMENTS**

## 前置

1. **激活 skill** `single-source-sync`(生成器纪律 + 口径级联的完整规则),把上面的参数透传给它;涉及"客户向 / 内部材料分层"的级联判断时,同时加载 `bid-playbook`。
2. **解析输入**:
   - 有口径变更描述 → 先做**爆炸半径映射**:列出该口径出现的全部位置(生成器脚本、数据源、叙事文档、构建脚本内嵌模板、兄弟文档、memory),逐个标注"公式驱动"还是"硬编码副本"。变更只落到**源**(脚本/数据),绝不直接改产物文件;派生值一律参数实算,禁止把算好的结果硬编码回去。
   - 无参数 → 纯一致性同步:跳过改源,直接从第 1 步跑校验链。

## 固定执行序(七步,顺序不可调换)

1. **lsof 写句柄检查**。重生成任何 xlsx/产物前,先 `lsof <产物路径>` 查办公软件写句柄(适用条件:macOS 上 WPS/Excel 等开着文件时,后台回存会覆盖刚生成的新文件——此坑实测复发 3 次)。**检出即停**:报告占用进程,请用户手动关闭且**不保存**,确认后才继续。
2. **手改检测(回捕)**。把现产物与"干净重生成版"做**逐格逻辑值 dump diff**(办公软件保存会重排内部 XML,raw diff 噪声大不可用):备份现产物 → 重生成到对比副本 → cell-dump 逐格对比 → 还原备份。检出用户手改 → **先停**:逐条捕捉手改意图,与用户确认后落进生成器源,再继续;绝不在"源与产物矛盾"状态下直接覆盖或提交。
3. **跑生成器**。按依赖顺序重跑全部受影响的生成器(成本 / 排期 / 报价等)。判据:命令零退出**不等于**产物已更新,必须走第 4 步实际验证。
4. **内容抽验**。对每个再生产物验证三件事:**新串在**(新口径值出现在预期单元格/段落)、**旧串亡**(旧值在该产物内全部消失)、**格式存活**(小数位、颜色标记等单元格格式未回退)。金额类表格还要复核"逐项可见加总 = 小计";注意小计对了明细行仍可能陈旧,须逐行核对。任一失败 → 回到源修,重跑第 3-4 步。
5. **全库 grep 残留**。用旧口径值/旧表述在整个仓库 grep(覆盖叙事 md、构建脚本、图 spec、README)。每个命中**逐条判读后再动手**:数字子串巧合(如搜 100 命中 1100)与"规则自述"(文档描述规则本身的句子)属合法误报,保留不改;确认为真残留的改**源**后回到第 3 步级联。批量替换后重读改动处上下文,确认语境仍通顺(表格用短标签、散文用全称的双规范不被破坏)。
6. **memory 核对 + 落决策**(固定末步)。核对 memory 中的数字/口径与交付物现状一致,过期条目更新;把本次口径变更的**决策与理由**写入 memory,防止后续会话按旧口径回潮。
7. **分组提交预览(不执行)**。按逻辑分组给出 commit 计划:每组列出**显式文件路径**(禁 `git add -A`)+ 提交信息草案 + **排除集**(无关的预存改动点名列出、明确不碰)。**只预览,不自动 commit**;用户逐组确认后才执行。

## 停止条件汇总

| 场景 | 动作 |
|---|---|
| lsof 检出写句柄 | 停,等用户关闭且不保存 |
| cell-dump diff 检出手改 | 停,捕捉意图落源后再继续 |
| 抽验发现旧串残留 / 格式回退 | 回源修复重跑,禁止手补产物 |
| grep 命中无法判读 | 列出上下文问用户,不擅自改 |
| commit / 覆盖含手改的产物 | 一律只预览,显式确认后执行 |

## 可选终检

级联触及 3 个以上文档时,建议追加 skill `adversarial-review` 跑一轮对抗校验,专挑"改了 A 处漏了 B 处"的跨文档口径不一致。
