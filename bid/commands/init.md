# /bid:init — 投标项目立项脚手架

**$ARGUMENTS**

## 执行流程

1. **激活 skill** `bid-playbook`,把上面的参数透传给它。无参数 → 从会话上下文推断项目名,推不出才问。

2. **判定线索状态**(三分支,决定后续走多深):
   - **新线索(未成单)** → 只建临时目录 `leads/<项目名>/`(内含 P0 问题清单 + `meeting/`),**不**搭正式双层结构。判据:没有中标/签约/明确排期,一律按新线索处理。成单后重跑 `/bid:init <项目名>` 转正,临时目录内容迁入正式结构——迁移动作只输出「旧路径 → 新路径」清单预览,不自动执行。
   - **全新成单项目** → 走第 3–6 步完整脚手架。
   - **既有目录要重组** → 先跳第 7 步审计,通过后再回 3–6。判据:目标目录已存在且非空 → 必走此分支,绝不直接覆盖。

3. **搭双层目录 + build 骨架**(客户向/内部硬隔离):
   ```
   <项目名>/
   ├── docs/           # 客户向交付物(对外,已脱敏)
   ├── docs/内部/       # 内部测算、口径依据、成本对比(绝不混入客户向)
   ├── build/          # 生成器骨架:数据源脚本 → xlsx/pdf/图,单一数据源
   ├── meeting/       # 会议纪要,按日期编年共享:YYYY-MM-DD-主题.md
   └── design/         # (可选)原型/视觉交接包
   ```
   判据:客户向与内部必须物理分目录;所有带数字的交付物今后必须由 `build/` 数据源生成(单一数据源原则,详见 skill `single-source-sync`),禁止手改产物文件成为事实源。

4. **生成 P0 问题清单** `docs/内部/00-P0问题清单.md`,预填 4 个必答项,每项带「状态: 未裁决/已裁决」+「裁决记录(日期 + 来源会议)」两栏:
   - **存量数据/案例授权**——这是第 0 步 blocking 项:未裁决前,禁止任何依赖存量数据的方案设计动工,不是可并行项。
   - **申报主体**——以谁的名义申报/备案,直接决定合规路径。
   - **真实规模基数**——DAU/用户数/并发用需求方给的口径,不自估(示例用教学值如 10000)。
   - **上线时间窗**——硬截止还是弹性,决定排期正排还是倒排。
   判据:4 项全部落入文件且状态列可追踪;缺任何一项信息 → 在清单里标「未裁决」,不要编造占位答案。

5. **初始化 memory 索引**:在项目 memory 目录建主题文件,并在 MEMORY.md 索引加一行(项目名 + 一句话摘要 + 核心口径占位)。判据:后续 `/bid:meeting`、`/bid:sync` 的固定末步会把口径决策落进这里防回潮——索引现在不建,它们到时无处可落。

6. **输出立项报告**(只报告,不 commit):目录树 + P0 清单 4 项状态 + memory 索引位置 + 下一步指引——开第一次会用 `/bid:meeting`;改任何对客数字用 `/bid:sync`;对外材料出稿前跑 skill `adversarial-review`。

7. **既有目录重组分支(审计先行)**:
   - 先 grep 全部构建脚本/生成器(`build/*.cjs`、`*.sh`、Makefile 等)里的硬编码路径,列出「重组后将失效的引用」清单。
   - 输出「移动/重命名计划表」:旧路径 → 新路径 → 受影响脚本及行号。**只预览,不执行**;等确认后逐条移动并同步修改脚本引用。
   - 判据:计划表里每条移动都标注了受影响脚本;遇到归属判断不了的文件 → 停下来问,不猜。

## 硬护栏

- **绝不自动 git commit**:只给出建议 commit message,执行权留给用户。
- **绝不覆盖已存在文件**:目标文件已存在 → 展示 diff 预览后停止等确认。
- 客户向目录中出现内部口径(内部成本中位数、内部代号、竞品对比)→ 立即 STOP 报告,这是分层泄露事故,不是小问题。
- P0 第 1 项(数据授权)未裁决期间,任何命令要生成依赖存量数据的方案章节 → 先提示回来裁决。

## 常用用法

| 场景 | 命令 | 效果 |
|---|---|---|
| 新线索刚接触 | `/bid:init 某To-B项目` | 只建 `leads/` 临时目录 |
| 成单后转正 | `/bid:init 某To-B项目` | 检测到 leads 已存在,出迁移预览 |
| 老项目目录重组 | `/bid:init 某To-B项目` | 检测到既有目录,先出路径耦合审计 |
