# 合规文案锁定清单与核对法

## 为什么锁定

AI 设计工具会自行编造**貌似合理**的界面文案填充原型。普通营销文案编了顶多难看;合规文案编了是违规/失效:免责声明措辞有法律含义、金融风险揭示句式受监管审定、标准化量表改一个字效度作废、危机干预话术与热线号码错了出人命级事故。这类文案是**不可替代的人工输入**,必须逐字进包。

## 锁定类别清单(逐项过,强合规行业全中)

| 类别 | 说明 |
|---|---|
| 免责声明/法律文本 | 「本服务不构成…」类;措辞有法律后果 |
| 金融风险揭示与收益表述 | 风险揭示书全文、「过往业绩不代表…」类监管审定句式 |
| 知情同意与隐私授权 | 医疗知情同意书、弹窗全文、勾选项文字、撤回路径说明 |
| 标准化量表/问卷题干与选项 | 逐字含标点与选项顺序;计分规则依赖原文 |
| 危机/风险干预话术 | 热线号码、转介措辞、升级流程提示 |
| 监管/备案要求的固定措辞 | 备案号展示、等级保护声明、审查过的宣传口径 |
| 未成年人保护提示 | 年龄门槛、监护人同意文案 |
| 法定名称与资质 | 主体全称、许可证/资质编号的展示格式 |

## 前置:授权与定稿

- 第三方版权内容(量表、测评工具)先确认**授权**再入包——授权没谈下来的内容进了原型等于公开演示侵权。
- 锁定文案必须是**定稿**:找需求方/合规口确认过再锁,锁了错稿比不锁更糟(错稿会被忠实复制到每一屏)。

## 包内标记法

逐屏简报里用显式标记块包裹,并在 master prompt 的「不可协商项」段重申:

```
【逐字锁定|禁止改写|禁止增删标点】
在开始前,请阅读并同意以下内容:……(全文)
【/锁定】
```

量大时单独立一个锁定文件(如 `04-locked-copy.md`),按屏索引;简报内只留引用标记,避免两处维护漂移。

## 生成后核对(每批必做)

对每段锁定文案在产物里做**全文精确匹配**,人眼扫一遍不算核对:

```bash
# 对产物 HTML 逐段 grep 锁定文案原文(注意转义);0 命中即打回
grep -rF "在开始前,请阅读并同意以下内容" <产物目录>/ || echo "MISSING: 知情同意全文"
```

- 匹配失败的常见形态:工具「顺手润色」了半句、繁简/标点被替换(全角逗号变半角)、长文被截断只渲染首段。
- 打回时在下一批 prompt 里点名该段「上批被改写,本批逐字恢复」,不要只重贴原文。
