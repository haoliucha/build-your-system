# 形态 A 示例模板:7 文件交接包

来自某 To-B 投标项目的实战包结构,接收方为「文件上传+知识库+稠密 prompt」型 AI 原型工具。**仅是示例**:文件数与命名按目标工具实测输入模型调整,不要照抄形态。

```
handoff/
├── README.md            # 包导览:各文件用途、喂入顺序、P0/P1/P2 分批放行计划
├── 00-简报.md           # 逐屏简报(生成层主内容)
├── 00B-主导色-demo.html # 主导色候选完整 demo(单文件,双击可开)
├── 01-视觉规范.md       # 精简视觉规范(原则级,不逐像素)
├── 02-参考板.md         # 视觉参考图板,每图带借鉴注记
├── 03-master-prompt.md  # 稠密结构化 master prompt(生成时的主输入)
└── tokens.css           # 双层设计令牌(护栏)
```

## 各文件写法

### README.md
写给接收工具的操作者(可能是另一个 AI 实例):哪些文件传知识库、哪个当 prompt 贴入、分批计划(P0 核心流程 N 屏 → 审 → P1 → P2)。没有导览的包会被按错误顺序消费。

### 00-简报.md(逐屏)
每屏一节,固定结构:**屏名/目标/全量真实文案/状态(空态/加载/错误)/交互要点/所属批次(P0/P1/P2)**。文案必须是真实定稿文案而非占位;合规锁定文案用标记块包裹(见 compliance-copy-lockdown.md)。这是包里最值钱的文件。

### 00B-主导色-demo.html
主导色不用色卡决策,用**完整界面 demo** 决策:同一屏做 2-3 个候选主导色的可切换版本,让用户在真实界面语境里选(色卡上好看的颜色铺满界面常翻车)。单文件、无外链依赖、双击可开。

### 01-视觉规范.md
只写原则:信息层级、字号阶梯、间距体系、圆角/阴影风格、图标风格。刻意保持薄——工具有自己的 craft,规范越厚越打架。

### 02-参考板.md
每张参考图附两行注记:「**借鉴**:导航密度/卡片节奏」「**忽略**:配色/字体」。没有注记的参考图会被整图照抄,连错误维度一起。宿主 App 的真实录屏帧直接入板(实测帧 > 任何转述)。

### 03-master-prompt.md
稠密结构化,分四段:
1. 角色与语境(产品一句话、目标用户、宿主环境)
2. 屏清单与本批范围(只列本批,其余标「后续批次勿生成」)
3. **不可协商项**:合规文案逐字使用、`--host-*` 令牌不越界、导航结构固定
4. 可自由发挥区(动效、插画风格、微交互)——明确授权比默认放任产出更好

### tokens.css(双层令牌)
```css
:root {
  /* ── 宿主层:来自录屏实测,不来自官方 VI ── */
  --host-accent: #3377FF;        /* 教学示例:实测点睛蓝(官方 VI 标称 #003366,App 实渲更亮) */
  --host-topbar-bg: #FFFFFF;     /* 二级页默认白顶栏,主题色仅点睛 */
  --host-tab-active: #3377FF;
  --host-tab-inactive: #999999;
  /* ── 自有品牌层 ── */
  --brand-primary: #6B4FA0;      /* 教学示例值 */
  --brand-surface: #FAFAFA;
}
```
双层令牌把「宿主 chrome(顶栏/tab/系统弹窗)」与「自有品牌区」物理隔离,评审时能分开讨论、宿主改版时只动 `--host-*`。

**适用条件(移动端嵌宿主 App 的 H5 原型)**:视口按 ~390px 设计;安全区用 `env(safe-area-inset-top)` 而非写死像素——写死的值在不同机型的宿主 webview 里会顶穿或留黑边。
