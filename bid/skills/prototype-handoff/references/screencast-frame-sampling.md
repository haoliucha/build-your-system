# 录屏抽帧 + 像素取样方法片段

> 跨 skill 共享的方法片段:原型交接(宿主视觉校正)与竞品调研(界面分析)共用同一套流程,两处引用、单处维护。

## 适用环境

macOS,`brew install ffmpeg imagemagick`(需 ImageMagick 7 的 `magick` 命令)。配套脚本:`scripts/extract-frames.sh`。

这里的 `scripts/...` 相对于上级 prototype-handoff 的 SKILL.md 所在目录解析,不是相对于本 reference 文件或进程 CWD。

## 第 1 步:索素材,附拍摄清单

向用户索要录屏时**随附拍摄清单**,一次录全,避免多轮补拍。清单模板(按需增删):

- 首页完整滚动一遍(慢速)
- 每个一级 tab 各停留 2 秒
- 二/三级页各代表性 1-2 屏
- 关键弹窗(同意/授权/分享)完整出现过程
- 深色/浅色模式各一段(如有)

分析产出的**文末单列「本次没拍到、待补拍」的模块**,给下一轮索要留清单。

## 第 2 步:固定节奏抽帧(禁用运动去重)

```bash
bash scripts/extract-frames.sh frames 录屏.mp4 frames/ 3   # 每 3 秒一帧
```

**反模式(实战翻车)**:用 `mpdecimate` 运动检测去重抽「变化帧」——滚动动画会击穿它,几十 MB 的录屏炸出上千冗余帧,完全不可读。固定节奏采样(2-4 秒/帧)产出可控帧数,信息密度足够。

## 第 3 步:contact sheet 两级阅读

```bash
bash scripts/extract-frames.sh sheet frames/ sheets/
```

生成带文件名标号的编号联络表(每张 6×6)。阅读纪律,**先地图后细读**:

1. 通读全部 sheet 建全局地图,记下关注帧的编号;
2. 只对文字密集/关键界面帧回原图放大细读。

直接逐帧读取上百张原图会烧穿上下文;sheet 先过滤能把细读量压到十几帧。

## 第 4 步:像素取样,校正视觉结论

```bash
bash scripts/extract-frames.sh pixel frames/f0012.jpg 100 40   # 输出该点 hex 色值
```

- **选稳定帧取样**:避开转场/滚动中的帧(有运动模糊与插值色)。
- 每个结论至少取 2-3 帧交叉验证;渐变取两端点。
- 区分记录「官方 VI 标称色」与「App 实测渲染色」两列——两者常不一致,**以实测为准**。
- 视频有色彩空间转换损耗,取样值做主色判定足够,做印刷级校色不够——标注精度即可。

## 第 5 步:提炼为纪律与令牌

取样结果不是留一堆色号,而是提炼成可执行结论,例如:

- 「主题色是**点睛**不是底色:二级页默认白顶栏,整条色带仅在个别页面出现」;
- 落成双层令牌:`--host-*`(宿主 chrome 实测值)与自有品牌层隔离(示例见 package-a-template.md);
- 真实参考帧挑代表性的直接放进交接包参考板——实测帧胜过任何文字转述。
