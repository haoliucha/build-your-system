# 改写与对抗复核协议

## 红线:信息零损失,只改外壳

改写只动修辞外壳,不动信息载体。不许动的清单:

- 排名/强弱标记(双勾 vs 单勾、评级符号等——它们是排序信号,不是装饰)
- 领域术语(哪怕长得像 buzzword)
- 数字、单位、金额、日期
- 引用与出处、法规编号、量表名
- 义务强度词(见下节)

**反模式**:批量去味改写把双勾/单勾排名标记压平成同级、把行业术语当 buzzword 删掉,独立审校才发现。→ 逐处还原,并把「信息零损失只改外壳」立为红线。

## 义务强度变更单独申报(与对抗审校流程的接口)

改写中出现「必须→建议」「应→可」「承诺→计划」级别的强度变化时:

1. 不得混入「语义等价改写」的自称里;
2. 单独列一张申报表(位置 + 原文 + 改后 + 强度方向);
3. 交作者逐条拍板,作者未确认前保持原强度。

这类变更有时确实该做(避免 overclaim),但决定权在作者不在改写者。

## 对抗 diff 复核(高权重,必做)

**反模式**:改写 agent 自称「语义等价」,对抗复核却抓出 4 处真问题——排序信号被压平、核心分析术语被系统性软化、生造缩略词、替换旧比喻时新引入拟物词,各占其一。

**操作**:
1. 复核者 = 与改写者**不同的实例/子代理**,不带改写会话上下文(避免自我辩护)。
2. 输入 = `git diff`(改写前先 commit 或 stash 出基线)。
3. 逐 hunk 专查四类:
   - **排名/强弱标记被压平**(✓✓→✓、加粗排序丢失)
   - **正当领域术语被系统性软化**(同一术语多处被换成泛化词)
   - **生造缩略词**(原文没有、行业也不通行的缩写)
   - **替换旧比喻时新引入的比喻/拟物词**(去了"大脑"来了"引擎")
4. 逐项回修;残留低价值命中按**导航价值**判定去留(帮读者定位的留,纯装饰的删)。

## 过度自证清理

同一防御性定语在正文重复 ≥2 次 → prose 层全删,证据只在表格内具体引用一次;免责尾巴全文只保留一处。

**反模式**:某防御性定语正文重复 3 次 + 三连防御短语叠在一句里,被审校判为「为撇清嫌疑而过度自证」——越解释越像有事。证据放进选型表/对比表引用一次,比 prose 里反复声明可信得多。

## 口述经历转写四动作

把口述素材(访谈、语音转写)转成对外文档的固定动作:

1. **重构成清晰子系统**:按逻辑重组,不按口述顺序;
2. **量化战绩前置**:数字、规模、结果放段首;
3. **术语按目标项目语境重定位**:口语叫法对齐到目标读者的行话;
4. **事实忠于原话零损失**:重组不改事实,不确定的专名按通用写法处理并**主动标注待确认**,不猜。
