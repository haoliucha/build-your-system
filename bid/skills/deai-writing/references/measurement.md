# 双层测量:正则毛统计 + 语义判读

## 为什么必须双层(高权重教训)

**反模式**:某 To-B 投标项目全套文档正则毛密度 70+/千字,若直接照单去味,会大量误伤正当学术引用与版式符号。逐文档语义判读后,净密度仅约毛数的**三分之一**。

**正确做法**:
- **层 1 正则毛统计**(`scripts/aiflavor-scan.cjs`):确定性、可复现、可前后对比。只当取证与基线用,不当验收指标。
- **层 2 语义判读**:逐文档扣误报 + 补录结构性 tell,产出净口径。净口径才是动刀依据。

语义判读可交给子代理执行,输入=扫描 JSON 的取证示例(带行号)+ 原文,输出=每类「真 AI 味 / 正当用法」的逐条裁定。

## 18 类正则 tell 逐类判读要点

按误报率分三档。**高误报类的命中默认先当正当用法,逐条证伪**;高信度类默认先当 AI 味。

### 高信度(命中基本可直改)

| 类别 | 抓什么 | 备注 |
|---|---|---|
| 元话语(综上/换言之/值得注意) | 篇章导航空转 | 超长文档少量真导航属正当,其余删 |
| "一句话"收束 | 口号式收束引导 | 几乎无正当用法 |
| 营销最高级(极致/碾压/遥遥领先) | 空洞最高级 | 投标文档中直删,用数据替代 |
| 这(就)是…的(关键/生命线) | 拔高式断言 | 常与段末升华句同现 |
| 强调填充(真正/本质/核心在于) | 无信息强调词 | 删后语义不变即删 |

### 中信度(看上下文)

| 类别 | 判读要点 |
|---|---|
| 破折号—— | 真解释性插入语属正当;承担"抖包袱"节奏的删 |
| 不是X而是Y / 既是…也 / 不仅…还 / 不做X做Y / 越X越Y / 从X到Y | 排比框架类:真实概念辨析、真实范围描述(如"从需求到上线")属正当;修辞性对仗删 |
| buzzword(护城河/飞轮/闭环…) | 部分词在架构语境是**领域术语**(底座、链路、端到端、一体化),逐条判读,勿系统性软化(见 rewrite-protocol) |
| 冒号式副标题 | 对仗式标题的正则近似;标题信息密度高则留 |

### 高误报(默认正当,逐条证伪)

| 类别 | 主要误报来源 |
|---|---|
| 装饰符(✓★◆▼→…) | 表格勾选、流程箭头、导航符是版式功能符;**双勾/单勾承载强弱排序信息,绝不可压平** |
| 中点排比(A · B · C) | 合法并列版式、外文人名点号 |
| 括号补充（…） | 引用、定义、法规编号、量表全称括注——这些是加分项不是 AI 味 |
| 粗体 | 术语强调、表头、关键数字 |

## 结构性 tell(正则抓不到,语义层必查)

1. **段末口号式升华句**:段落收束时拔高到愿景/价值观。删除性价比最高。
2. **对仗/排比式标题**:整齐四字对仗、冒号副标题成排堆叠。
3. **拟物化/道德化词汇**:把系统说成生命体("大脑""心脏"),把工程选择说成品德。
4. **过度自证**:同一防御性定语正文重复多次(见 rewrite-protocol.md)。

## 脚本输出解读

- 密度 = 命中数 / 千 CJK 字。HTML 先剥离 `<style>/<script>` 与标签再计 CJK;HTML 粗体按 `<strong>/<b>` 标签与内联 `font-weight` 计;装饰符不数 CSS 内符号。
- 每类取证示例前 3 条带行号,供语义判读定位。
- 前后对比:动刀前后各跑一次。毛密度下降只是参考——**验收看语义层净口径 + 信息零损失复核**,不看毛数归零。
