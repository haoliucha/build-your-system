# bid-costing — 可辩护的成本测算

管「成本怎么算得可辩护」。谈判层的报价结构归同插件 presales-tactics,见文末边界。

## 核心心智:报价 = 参数化模型的输出

把全部成本数字收进一个参数化生成源(脚本/表驱动),xlsx/md 只是渲染产物。改报价 = 改参数重算并全链路级联,绝不手改产物数字;删档/删单值后 grep 全仓确认旧数消失。凑整与否由决策者拍板,凑整后回推可辩护参数,不留无来源的数。

## 五条纪律

### 1 价格阶梯:列表价是唯一折扣基准
任何单价先定口径:官网列表价 / 活动价(首购、新客) / 包年承诺价 / 按量价。折扣只能乘在列表价上、只乘一次;活动价本身已是折扣,再乘 = 双重打折,得出根本拿不到的基线。限购一件的首购价盖不住多实例生产规模,不作基线。二手折扣标签必须算术反算互证(价格÷标称折扣应还原出一致列表价),对不上整行撤回。溯不到源的数字当场撤回,永不落盘。

### 2 依据分层:每行标四级依据 + 置信度
确定算式 > 官方 list 价 > 人力薪酬行情 > 估算带。逐项落档,Σ = 小计可视加总;查不到官方价宁标「估算/低置信」也不造假精确数。被质疑头条数时双管齐下:溯源原始出处 + 自底向上逐件重构;两者同量级则常量不改、证据落档——补证据不等于改数。

### 3 降数字走需求侧
客户嫌数字大,下调规模假设,不上调能力参数。容量参数须有公开 benchmark 可辩护,经不起推敲就回滚并标「待 POC 压测」。凑不出目标数就报诚实值 + 差距 + 补齐条件;用户给数字目标时先确认其实现机制再动手。宁诚实偏高,不报上线会穿底的低价。

### 4 范围两端同情景配对
低~高 = 全低位情景 ~ 全高位情景,禁混口径;跨方案对比同情景同会计口径,拆分方式造成的差额标「口径差非真实节省」。免/不免二元情形不是区间:收敛基线值,触发条件入备注。阶梯扩容的斜率微差是伪精度,统一取整保守值。一张表禁两轴叠加。附「未计入项」清单;可选项 = 不进合计的 +Δ 行;「X 万起」改区间 + 典型值。

### 5 关键输入用户拍板,拍板前冻结
影响全表的单价/折扣档/规模档是业务决策:呈现「选项 × 下游 TCO 影响」对照表等拍板,拍板前不动任何数字,防按错价白算一轮。

## 专项推导链(细节见 references/capacity-and-labor.md)
- **算力**:模型参数→权重显存→KV cache 每路→单卡并发→实例→单价;驱动变量是峰值并发(≈注册规模 3–5%)而非注册数。弹性容量按「按量单价 × 日峰值时长 × 365」建模,不按 24×7 包月;保底容量才走包年折扣。
- **人力**:团队与工期锁定后 = 确定值(单价 × 人数 × 月),效率省日历不省人月,区间只留真浮动项;利用率系数用排期实证装载锚定;计费口径与排期日历可有意分层,但须表内注释调和并向决策者声明。
- **合规**:角色法定刚性不可省不可合并,可弹的是薪酬档位与承担方;转移出去的成本红标 + 脚注,防误读为真省。每条一次性成本对照排期工作包,防重复计价。

## 反模式速查(真实踩过)

| 反模式 | 正确做法 |
|---|---|
| 把活动价当列表价再打折 | 以一手截图/官网价为锚重建价格阶梯 |
| 照搬二手「8 折」标签 | 反算发现实为 4.2 折,整行撤回 |
| 上调单卡并发凑卡数 | 回滚 + 下调需求档 + 标待压测 |
| 给锁定人力设「高效低位」 | 删无依据低位,锁确定值 |
| 免/不免写成连续区间 | 基线值 + 触发条件入备注 |
| 已含在排期的工作再单列费用 | 删行 + 备注外采/人力边界 |
| 规模档 × 低高区间两轴叠加一张表 | 主表锁单一基准,规模档单独成曲线子表 |

## 脚本
`node scripts/discount-check.cjs [声称列表价|-] <价格:折扣>...` — 折扣标签反算互证:各路径反算隐含列表价,偏差超阈值(默认 0.5%)即 FAIL。

## 环境提示(macOS + WPS/Excel 适用)
xlsx 重生成前先 `lsof <文件>` 确认无办公软件持有写句柄,否则回存会覆盖新产物。用户手改 xlsx 后:先备份,用 ExcelJS cell-dump diff 捕捉改动(WPS 会重排 XML,raw diff 噪声大),再把改动落回生成源。
