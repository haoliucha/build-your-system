# 报价锚定与分档结构

所有案例均来自「某 To-B 投标项目」红队复盘,已抽象;示例数字为教学值。

## 1. 摸底问题清单(开谈第一步)

报价锚 = **规模 × 频次 × 人数**,不是技术难度。起草任何报价/话术前,先确认:

| 维度 | 问什么 | 为什么 |
|---|---|---|
| 人数 | 使用者/审批者/管理员各多少人 | 直接决定档位与话术剧本 |
| 频次 | 目标流程每月发生多少次 | 低频客户对"效率"话术无感 |
| 规模 | 组织总人数、站点/分支数量 | 区间上探的明示触发条件 |
| 现状 | 现在用什么工具/怎么凑合的 | 真实竞对常是"免费+现状凑合" |
| 预算感 | 有没有报批流程、决策人是谁 | 决定第 0 期怎么设 |

**反模式 → 正确做法**:初稿话术按"十几个使用者的团队"剧本写,套到极小体量客户身上,"省人力/月底催办/多级审批"被当场戳穿"你说的我都没有",之后所有话打折听。→ 摸底先行,对极简客户换成他真在乎的价值口径(如省心、留痕),而不是套模板。

**体量差的量级感**(教学值):同一需求,10 人小团队合理报价若是 100,那 1000 人多站点组织可以是 1000–5000——差 10-50 倍是常态,不是漫天要价;反过来把大组织的价格锚报给小团队,就是"另一个星球的数字"。

## 2. 三档锚定设计表

| 档位 | 角色 | 设计纪律 | 反模式 |
|---|---|---|---|
| 保底档 | 台阶,不是目的地 | 必须"做残":只覆盖能跑通的最小闭环,**不包含主推档的真痛点功能** | 保底档把两个最痛的功能全包了 → 客户踩上台阶就不走,只买保底甚至要白送 |
| 主推档 | 独占痛点,承接 80% 成交 | 客户摸底出的真痛点功能全部收在这一档 | 痛点散落各档 → 主推档失去存在理由 |
| 高档 | 锚,不是商品 | 只在客户**主动问**时报,且主动说"你这体量用不上、不建议" | 对小体量客户主动报高档 → 反让客户怀疑你不懂他的盘子,连带摧毁整个定价可信度 |

## 3. 干净数字与区间纪律

- 确认体量后,对客只报**一个干净数字**(或极窄区间)。
- 区间宽度纪律:初稿区间宽达 2.7 倍被红队判"自己都不知道值多少钱"。区间上探必须绑定**明示触发条件**——"超过 N 人 / 每月超过 M 笔 / 多站点则上浮至 X",让客户看见价格随他的量走,而不是随谈判走。

## 4. 砍价降级树(先成文,再上桌)

被砍价时按顺序走,**每一步砍的是范围,不是量级**:

1. **第一刀:砍功能范围** → 给"主推档减去某模块"的中间档,价格降幅与砍掉的范围对应(如 1000 → 800)。
2. **第二刀:后置阶段** → 二期功能移出首期合同,首期金额下来、总盘子不缩。
3. **最低档**:只在客户**明说预算极低**时才掏,不主动展示。
4. **永不砍**:安全合规基线(见 redlines-and-data.md)。

**反模式 → 正确做法**:异议话术从 1000 一步降到 100 → 客户立刻明白是漫天要价,定价信誉崩。→ 任何单步降价不破量级(教学值:降幅 ≤30%),且每次降价必须对应一次范围削减,"白降"零次。

## 5. 三层口径详表

| 层 | 内容 | 给谁看 | 纪律 |
|---|---|---|---|
| ①市场价值锚 | 同类采购的市场价/官方列表价 | 材料背书、内部论证 | 只作量级参照,**不直接当报价甩给客户** |
| ②阶段建议价 + 内部底线 | 各阶段建议报价、最低可接受线 | 团队内部 | 绝不出现在对客材料;底线成文,谈判者不临场发明 |
| ③对客口径 | 只报量级 + 低门槛启动方案 | 客户 | 干净数字;第三方硬成本(云资源/官方产品订阅等)**单列透传**,注明"官方列表价,非我方利润项" |

- **第 0 期**:小额诊断/POC/原型(教学值:主合同的 5–10%),用途是降成交门槛 + 换取真实数据摸底,产出物对客独立可用,即使不签后续也不亏。
- 三层混用的典型事故:把①的市场锚当报价 → 吓跑;把②的底线提前露出 → 谈判空间归零。

## 6. 竞品价格情报纪律

- 竞品与价格情报**实时检索**,不凭记忆。
- 询价制厂商的第三方测评价格只作量级参考,标注"勿当确切报价引用"。
- 平台配额/价格类数字注明口径年份,签约前再核一次。
