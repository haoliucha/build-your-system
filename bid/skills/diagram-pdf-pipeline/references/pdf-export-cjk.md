# 中文 PDF 导出细节

本文承接 SKILL.md「导出」一节。核心事实:中文文档导出 PDF 的坑大多是字体与渲染引擎的组合问题,且往往在源文件与 HTML 预览里完全看不出来。

## 1. CJK-first 字体栈:根因机理

**症状**:导出 PDF 里全角括号变弧线、冒号句号变上浮小点,但正文汉字完全正常。因为「大部分文字都对」,极难第一时间定位。

**根因**:字体匹配是逐字形(per-glyph)的。Latin-first 字体栈(如 `font-family: Helvetica, "Some CJK", sans-serif`)下:

- 汉字在西文字体里没有字形 → 回退到后面的 CJK 字体 → 渲染正常;
- 全角标点(U+FF08 全角括号、U+FF1A 全角冒号等)在部分西文字体里**有**字形 → 直接命中,取错字形 → 只有标点坏。

**引擎差异**:Chrome 的回退策略较健壮;部分排版引擎(如 Prince 一类)不会正确回退,同一份 CSS 在不同引擎下结果不同。修复路线二选一:字体栈改 CJK-first,或整条管线换 Chrome 渲染。

**修复要落到源头**:不只重出当次文件——把 CJK-first 栈写进全局导出配置/主题 CSS 源头,让未来所有导出(含原引擎路径)在源头就正确。定位时先做隔离复现(最小 HTML + 两种字体栈对照)证明根因,再动配置。

**引擎选型列 trade-off**:书签大纲支持、文件体积、水印、标点正确性——按交付物实际需要选,不默认单一引擎。需要书签时保留支持 outline 的引擎 + 修复后的样式。

## 2. headless 环境:先探测可用 CJK 字体

**症状**:headless Chrome 导出的 PDF 正文中文整页空白(只剩标点数字)。根因:系统惯用的 CJK 字体名在该 headless 环境解析不到,`sans-serif` 兜底又落到无中文字形的西文字体。

**做法**:

1. 写一个字体探测页:同一段「中文样本+全角标点」用每个候选字体名各渲染一行,与一行故意用不存在字体名(必然走兜底)的做宽度对比或截图对比,逐一实测哪些字体名真正可解析。
2. 字体栈:实测可用的 CJK 字体打头 + 实测能兜中文的通用族收尾。绝不让回退落到无 CJK 的西文字体。
3. 字体本地化:去掉外链 CDN 字体依赖——外链拉取曾直接卡死导出。
4. 适用条件:macOS 桌面机上系统字体名通常可用,此步是保险;Linux CI/容器环境必做,字体集差异大得多。

## 3. mermaid 禁令:根因与替代

**翻车原样**:自检 HTML 预览通过后交付,用户随手翻 PDF 就发现**所有架构图的中文节点标签整体丢失**(图形还在、字全空)。根因:浏览器打印 PDF 时 mermaid 渲染的 SVG 文本不内嵌 CJK 字体,打印管线取不到字形。

**规则**:交付物配图一律不用 mermaid。改用可栅格化管线:drawio 制图导 PNG,或程序化 SVG → PNG 内嵌(见 `diagram-pipeline.md` 第 3 节)。HTML 预览正常不能证明 PDF 正常——这是本条最阴险之处。

## 4. 嵌图内联 2x PNG,不用带滤镜 SVG

**翻车原样**:精排 HTML 导 PDF 连续报 Printing failed,唯一输出是一个裸 `}`;先后误猜 headless 模式、中文路径等方向,均不中。

**二分定位法**(导出反复失败的标准流程):

1. 极简 HTML(一行文字)验证打印链路本身可用;
2. 把文件复制到纯 ASCII 路径,排除路径编码问题;
3. 对 HTML 内容做二分删减,锁定崩溃源。

该案例锁定:内嵌 SVG 里的 blur `filter` 是崩溃源。**全部换成内联 base64 2x PNG 后一次成功**。SVG 与多页 drawio 保留作可编辑源,只是不进打印面。

## 5. 书签/大纲

Chrome 打印引擎(printToPDF / Cmd+P)导出的 PDF 天然没有 outline 书签。取证方法:`pdfinfo` 看 Producer 同为 Skia/PDF,有无书签的差别只在是否跑过注入/导出脚本。

**首选路径**:用 `scripts/add-outline.cjs`(playwright-core + 系统 Chrome)重新导出,`page.pdf({ outline: true, tagged: true })` 从 h1-h6 自动生成层级书签;脚本固化进构建链。

**备选路径**(为已有 PDF 按标题文本搜索注入书签)的两个必踩坑:

- **只前进的有序游标**:按标题出现顺序搜索,游标只向前移动,每条书签在游标之后找首次命中。否则:短标题(如「附录」)会撞上前文的交叉引用(「见附录」)错锚到前页;首章会因与目录同页被整页排除而丢失。允许书签与目录同页。
- **字号消歧**:同名文本取字号最大的匹配,避免目录页同名行被误认为真实标题。
- 注入后做页码递增校验 + 逐条书签点击验证。

## 6. CommonMark 全角边界:强调标记失效

Markdown 里紧贴全角字符的 `**加粗**`(如紧贴中文括号)可能不被 CommonMark 解析,渲染结果漏出字面星号。构建链加一道后处理:全文扫描残留的 `**` 等标记符,补成对应格式;重导后全文验证无残留。

## 7. 版面:竖屏截图入 A4

手机竖屏比例截图放进 A4 文档,按页宽铺满必然过高 → 跨页割裂。做法:限制最大高度约半页并居中,达到 2 图/页、图文同页的报告观感。尺寸需目检迭代 1–2 轮定稿(先大后小)。
