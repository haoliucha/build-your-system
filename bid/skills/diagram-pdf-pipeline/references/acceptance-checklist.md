# 验收清单

本文承接 SKILL.md「验收」一节。核心原则:**构建成功、grep 清零、代码无报错,都不能替代亲眼看渲染结果**。用户曾原话:「PDF 文件需要重点检查,我随便看一下就发现里面的图渲染的不对」。

## 1. 先排除静默失败(产物新鲜度)

导出链路某一环静默失败会让旧 PDF 冒充新版。真实翻车:printToPDF 报错只输出一个裸 `}`,PDF 停在旧版,直到 PDF 页面出现**源文件里已删除的旧图题**才暴露。

每次导出后:

- [ ] 逐字核对导出日志的成功行(不是「没看到报错」,是「看到了成功行」)
- [ ] 抽验一处**本次新改**的文字/图确实出现在 PDF 中
- [ ] 发现 PDF 内容与源 HTML/md 不一致时,第一怀疑对象是构建陈旧,而非源文件写错

## 2. 逐页人工目检项(终稿必做)

打开并检查渲染后的 PDF,逐页/逐图核对:

- [ ] 中文完整:无整段空白、无缺字(mermaid/字体栈问题的最终防线)
- [ ] 全角标点正常:括号/冒号/句号无变形上浮(CJK-first 栈问题的最终防线)
- [ ] **边线路由**:无横穿框体、无误连(多节点汇入同一框要专门核对汇入点)——生成器死三元、坐标推断类 bug 生成代码跑通也不报错,只有肉眼能兜
- [ ] 标签不被色块遮挡、不挤窄缝
- [ ] 图不跨页割裂(嵌图所在页 + 相邻页都看)
- [ ] 无重复标题、图题与正文引用一致
- [ ] 书签(如有):逐条点击验证页码,标题层级正确

## 3. 页码漂移:以渲染为准

**反模式**:用 pdftotext 按换页符计数定位「图在第 N 页」——计数会漂,报出的页码可能实际是另一张图。

**正确做法**:直接渲染目标页及相邻数页为图片逐页目检,确认图完整落页。文本工具只用来缩小范围,结论以视觉为准。

## 4. 图缺陷修复闭环(五步,不跳最后一步)

1. 按用户截图/自己目检结果逐张定位问题
2. 读生成器源码找**几何根因**(band 间隙/行起点避让/自动折线贴边/死三元)
3. 改 spec(不是改产物文件)
4. 重渲**全部**产物(SVG/PNG/可编辑源同源重出)
5. 逐张目视复核确认——修好一处可能碰坏另一处,这一步绝不跳

## 5. 结构性自检(补充手段,非替代)

- grep 可用于结构性规则自检(如引用编号连续、无残留占位符、无泄漏的内部措辞),作为目检的**补充**。
- HTML/PPT 类视觉交付物同理:真实渲染逐页截图核对 + grep 结构自检双管齐下。

## 6. 精修分级

- 对外正式交付图:像素级精修,走上述完整闭环。
- 内部文档草图(如 ASCII 图):CJK 等宽参差是工具固有限制,连线逻辑正确即可,不做脆弱的手工重排——投入与产物等级匹配。
