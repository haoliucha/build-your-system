# 出图管线细节

本文承接 SKILL.md「出图」一节,展开选型、生成、转义与编号纪律的操作细节。所有案例来自某 To-B 投标项目,已抽象化。

## 1. 方案选型:全网对比调研取优胜者

**反模式**:发现手头有一个用过的图表生成器,直接 fork 铺开全量。曾被叫停:「再去做一次全网调研,对比调研出来的最佳方案与既有方案,取优胜者」。

**流程**:

1. 列候选:mermaid、D2、graphviz、drawio(GUI/CLI)、程序化 SVG 生成器等,连同内部既有方案一起进对比表。
2. 对比维度:CJK 支持(能否内嵌/栅格化中文)、可栅格化为 PNG、多页支持、自动布局能力、可编辑源格式、是否依赖 GUI/CLI 可用性、迁移丢失项。
3. 逐项列证据后取优胜者;允许 Hybrid(该项目结论反直觉:内部方案的风格层已达标,只需替换布局层)。
4. 涉及丢失项的取舍,列明选项交用户拍板,不替用户决定。

**单图试点再批量**:选最复杂的一张图端到端跑通(spec → SVG/PNG → 嵌入文档 → 渲染 PDF 对应页目检),锁定模板与参数后再并行铺开其余。批量后仍逐张 Read 渲染结果核验——曾在批量产物中抓出一张拓扑图被自动布局画成串行链,需手工重排。

## 2. 风格层与布局层分离

图的品质可拆成两个独立问题:

| 层 | 负责什么 | 典型缺陷 | 修法 |
|---|---|---|---|
| 风格层 | 配色、圆角、字体、图例、导出格式 | 丑、不统一 | 复用已验证的风格引擎 |
| 布局层 | 节点坐标、连线路由 | 连线交叉、横穿框体、标签遮挡 | 换自动布局引擎(如 elkjs) |

「连线与内容交错重叠」是布局层问题,不要因此整体换工具——那会丢掉已验证的 CJK/矢量/多页能力。

**分图型选布局策略**:

- 流程图/拓扑图:自动布局(elkjs 等),根治交叉。
- 分层图/分区图:按 band 手摆——每层一条横带,构造上零重叠。

**手摆图的几何根因清单**(修复时读生成器源码对照):

- band 间隙过窄 → 标签被色块遮挡
- 行起点未避让标签宽度 → 标签挤进窄缝
- 自动折线贴边 → 连线与框边重叠
- 生成器死三元(两分支同值)→ 全部节点误连同一个框;自动推边把节点横向穿过其他框

这类 bug 生成代码跑通、整体不破版,只有肉眼能兜——见 `acceptance-checklist.md`。

## 3. 单一代码 spec 同源出多格式

环境无 drawio CLI 等 GUI 工具可依赖时,确立策略:

- 每张图定义为一份代码 spec(数据 + 几何参数)。
- 脚本从 spec 同源生成三件套:展示用 SVG、文档内嵌用 2x PNG、可编辑源(如多页 drawio 文件)。
- 改图只改 spec,一键重出,三格式永远一致。
- 若有人在 GUI 里手改了产物文件:用语义级 diff(逐 cell/逐节点 dump 后对比,不看 raw XML diff——GUI 会重排 XML 产生大量噪声)捕捉改动,反向移植回 spec 再重出。**绝不让 GUI 文件成为数据源**,否则下次重生成会静默覆盖手改。

## 4. XML 属性内富文本:整体转义 + 合法性校验

**反模式**:把节点描述的原始 HTML(含 `<b>`、`<br/>` 等)直接拼进 drawio 的 `value="..."` 属性——属性值里的裸 `<` 和引号使整个 XML 非法,文件打不开。

**正确做法**:

1. 先拼完整 HTML 字符串,再**整体转义**(`<` → `&lt;`、`"` → `&quot;` 等)写入属性;渲染端靠 `html=1` 标志解码。
2. 交付前跑 `xmllint --noout <file>` 校验合法性,进构建链。
3. 排障提示:文件打不开时别停在「引号问题」的第一直觉——该案例真凶是裸 `<`,要深挖到真根因。

## 5. 编号与单一事实源纪律

**插入新图**:向已有「图 1…图 N」体系插图,选纯新增编号(如「图 0」),绝不 renumber。重编会级联破坏:正文多处嵌入、图索引、其他文档对图号的硬引用。以选项形式让用户在「纯新增(推荐)」与「全量重编」之间拍板,选定后零 renumber 落地。

**近似总览图收口**:图集里出现两张相似但不一致的总览图时——

1. 逐元素核实内容关系;若一张是另一张的严格超集,以超集为准。
2. 退役旧图:删 spec + `git rm` 全部产物(SVG/PNG/可编辑源页),两处文档统一引用同一张。
3. 跨文档共享的图,标题措辞保持中性(不绑定单一文档语境)。
4. 接受轻微编号前缀瑕疵(如退役后图号不连续)作为单一事实源的代价——比 renumber 的级联破坏便宜得多。全仓库同一主题只留一张架构图。
