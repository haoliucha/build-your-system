# 多 agent 审校编排细节

执行者假定为支持独立执行单元的宿主编排者(spawn 并行子代理/后台任务)。本文补全 SKILL.md「工作流实施」一节的程序性细节。

## 1. 编排拓扑

```
执行者自修(机械可检问题清零)
  → 并行扇出 N 个透镜子代理(互不通气,各领一个角度)
  → 裁定人收敛(多方共指置顶/用户事实优先/机械问题退回)
  → 逐条修复
  → 重跑对应透镜验证(不以「已修」自述为准)
  → commit / 交付
```

- 透镜数量:重要交付物 ≥3 个不同视角(如产能算术/范围依赖/评标呈现,或一致性/脱敏/overclaim/准确性)。
- 高优先发现被多方共指时基本必为真问题;单透镜独报的判断类发现交裁定人权衡。
- 跨项目/跨文档综合对比不凭记忆写:先派并行 agent 重读双方原文提取事实,再派独立核验 agent 对抗核对归因与数字——核验常顺带揪出交付物自身的陈旧 bug(实战一次捞出 4 处陈旧值)。

## 2. 子代理简报四件套(逐件说明)

结果质量由简报质量决定,四件缺一不可:

1. **锁定基线常量**:对客锁定口径、已锁决策、用户已明确事实。写明「与本区块冲突的发现标为待核实,不要求改稿」——否则子代理会拿着样本推断顶撞真实口径。
2. **严格输出 schema**:便于直接落表/机读收敛。**属性键只用 ASCII**——中文属性键会让全部 agent 启动即被 API 400 拒绝(`Property keys should match pattern`),中文语义放 description 或值。这是实测翻车,不是风格偏好。
3. **来源优先级**:官方 > 一手实测 > 社区 > 代理商转述。没有这条,子代理会把博客转述当官方口径。
4. **反编造纪律**:查不到写 unverifiable+核验路径;每条发现带可核验锚点;被墙即报告不绕编。

## 3. 锚点样例先行(批量同构产物)

批量生成同构产物(多份透镜报告/多页图/多份文档)时:

1. 先**手写 1-2 个锚点样例**,锁死全部约定:目录结构、命名、样式/引用方式、schema 字段。
2. 把锚点作为模板喂给并行 agent,按 **author→verify 成对**生产其余(作者代理产出,验证代理对照锚点检查约定符合性)。
3. 最后统一集成,跑**真实验证**(渲染/编译/脚本),不以各代理自报为准。

跳过锚点直接扇出的代价:每个代理各自发明约定,集成时全量返工。

## 4. 检查器有效性反向验证(程序)

任何「检查通过/清零」都要过这三步才可信:

1. **正向**:检查器对目标文件跑出 0 命中/绿灯。
2. **反向**:向目标文件(或副本)注入一个已知错误(如脏词 `FORBIDDEN_TOKEN_100`),重跑,**必须**抓到;抓不到=检查器空转(路径错/参数未展开/管道吞码)。
3. **复原**:删除注入,复测干净。

配套纪律:

- 检查命令**报错≠通过**。多文件变量未 word-split 被当单一路径,检查实为报错——必须用显式文件列表重跑拿真实结果。
- `grep … | head` 会掩盖 grep 退出码造成假阴性;清零判据用 `grep -c` 计数=0(`scripts/check-residuals.sh` 已固化,含 selftest)。
- 命中要人工分类:「禁止出现 XX」的规则自述不是泄露;验证报告说明每处命中性质。

## 5. 送达语义与进度

- 子代理任务完成判据 = 结果**已通过消息通道送达编排者并确认发送成功**(如 SendMessage 返回 success),不是本地会话产出了文本。产出与送达是两件事。
- 编排者中途询问进度时,子代理返回**当前最佳结果**,不为追求完美而拖延。
- 编排者收齐后对子代理发正式关闭指令,闭环。
- 红队/审校可后台并行跑:期间先把最可用的片段给用户,结果回来后折进定稿,不阻塞主线。

## 6. 被质疑与「只记录不改」通道

用户质疑交付物某论断时:

1. 先给**诚实初判**(包括承认站不住),不辩护。
2. 跑多路联网取证 + 对抗裁定坐实。
3. 产出多档改写口径(保守版/带锋芒版)由用户定调。
4. 用户选「先不改,只记录」→ 建待办(记录改动位置与判级依据)+ 写 memory 防重犯,交付物一字不动。

## 7. 环境相关的坑(适用条件已标注)

- **办公软件回存覆盖**(适用:macOS + WPS/Excel 等打开着目标 xlsx/docx 时):重生成文件前先 `lsof <文件>` 检查写句柄,有占用先让用户关闭——否则办公软件稍后回存会覆盖新产物。
- **用户手改过的表格捕捉**(适用:办公软件保存过的 xlsx):办公软件保存会重排内部 XML,raw diff 噪声极大;用单元格级 dump(如 ExcelJS 逐格导出)做 diff 捕捉手改,流程=先备份→按源码重生成一份对照→逐格 cmp。手改常自相矛盾(改了主表漏了图表/结论),捕捉后落回生成器源码而非直接 commit 手改产物。
- **grep 兼容性**(适用:macOS BSD grep 与 GNU grep):本 skill 脚本只用 `-c/-n/-E`,两者语义一致;避免依赖 GNU 专属选项(如 `-P`)。
- **登录墙/风控站点抓取**(适用:外部取证):被拦即停下报告或改用已验证既有材料,不围绕编造;不要因单次 403 就断言「页面不存在」。
