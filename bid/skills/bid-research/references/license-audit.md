# 开源选型 license 穿透审查

## 核心认知:三层分离

一个开源 AI 项目里,**代码、权重、数据集是三份独立的许可**,任何一层的宽松都不代表另一层可商用:

| 层 | 常见位置 | 常见坑 |
|---|---|---|
| 代码 | 仓库根 LICENSE 文件 | 仓库标 Apache-2.0/MIT,但它**只覆盖代码** |
| 权重 | 单独的 MODEL LICENSE / 社区许可 / 模型卡 | 商用规模阈值(如月活超过某数需另签)、用途限制、再分发条款、输出物归属 |
| 数据集 | 数据集卡片 / 来源声明 | 「仅限研究用途」的数据集,训出的权重商用性存疑 |

## 继承链:微调权重是衍生物

微调权重继承**底座模型**的 model license——哪怕微调方把自己的仓库标成 Apache-2.0,也改变不了继承关系。审查时沿继承链一路向上追到最底层底座:

```
候选微调模型 → 其底座 → 底座的底座(如有) → 每一层的 model license 逐个读原文
```

任何一层带商用限制,整条链的商用判定就取最严的那层。

## GPL 开核(open-core)识别

套路:开源版用 GPL/AGPL(传染性协议),商业闭源版另售。识别信号:同一产品官网同时挂「社区版 GPL + 企业版商业授权」;README 里「商用请联系」。对 To-B 私有化交付,AGPL 的网络分发条款尤其危险——别只看「开源」两个字就选型。

## 判定决策树

```
候选组件
├─ 只用代码(自训权重)? → 审代码 license 即可(仍要看数据集)
├─ 要用权重?
│   ├─ 沿继承链逐层读 model license 原文
│   ├─ 有商用阈值/用途限制/再分发限制? → 标「受限」,把限制条款原文摘进报告
│   └─ 底座任何一层不可商用? → 权重整体「不可商用」
├─ 要用数据集? → 单独审数据集许可;「研究用途 only」→ 训练产物商用「存疑」
└─ GPL/AGPL 开核? → To-B 私有化交付默认排除,除非买商业授权
```

## 输出格式

选型报告中每个候选组件一行:

| 组件 | 层 | license | 继承自 | 商用判定 | 依据(链接) | caveat |
|---|---|---|---|---|---|---|
| 示例模型 | 权重 | 社区许可 vX | 底座模型 A | 受限(月活阈值) | 许可原文链接 | 阈值条款需法务确认 |

商用判定只允许四值:**可 / 受限 / 不可 / 存疑**。「存疑」必须附核验路径。

## 常见结局提醒

穿透审查跑完,常见结局是:**真正可商用复用的只有数据集与方法论(训练配方、评测集),权重反而不行**。选型报告要把这个区别写清楚——「借鉴其方法论 + 自有底座复现」与「直接用其权重」是两条法律风险完全不同的路线,别让读者混为一谈。
