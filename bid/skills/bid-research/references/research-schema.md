# 结构化调研输出 schema

## 六字段 schema

每条进入交付物的量化结论,按六字段落纸,缺一不可:

| 字段 | 含义 | 要求 |
|---|---|---|
| 区间 range | 低~高 | 永远给区间,不给孤立单值 |
| 典型值 typical | 区间内最可能的值 | 说明为什么典型(众数?中位?官方推荐配置?) |
| 主驱动 driver | 什么变量在推动这个数 | 一条结论一个主驱动,多了说明还没想清楚 |
| 来源 source | 分四档:官方 list 价 / 公开报道 / 行业行情 / 估算 | 带链接或算式;估算必须给推导 |
| 置信度 confidence | 高 / 中 / 低 | 一句理由(如「确定算式,低=高」「单一来源未交叉」) |
| caveat | 已知不覆盖什么 | 显式写,不留给读者猜 |

示例(教学数字):

> 某组件月成本:区间 100–200 元,典型 150 元;主驱动 = 实例规格;来源 = 官网列表价(链接);置信 = 高(确定算式);caveat = 未含流量费与突发扩容。

## 金额定性规则

- 联网检索到的第三方金额,**只取定性**(量级、趋势、相对高低),不当精确输入进任何成本/报价表。
- 能进报价表的数字只有三种:官方 list 价(带链接)、可复算的算式(展示算式)、显式标注的行情区间(标「行情」档)。

## 留白格式

查不到就留白,禁止编造或含糊带过。留白必须附核验路径:

> 查无公开数据。核验路径:① 向官方销售询价;② POC 阶段实测;③ 参考同类组件 X 的公开定价做上界。

## 三角验证登记表

重大技术路线主张,用登记表强制多链互证:

| 主张 | 证据链 1 | 证据链 2 | 证据链 3 | 独立性 | caveat |
|---|---|---|---|---|---|
| (一句话) | 类型+来源+作用域 | 同左 | 同左 | 三链为何互相独立 | 各链不覆盖什么 |

- 典型组合:**学术实证(同行评审)+ 法规条款(可溯源/可删除等合规要求)+ 业界工程实践(头部开源项目的实际组合用法)**。
- 每条链要标**作用域**:论文证的是效果、法规证的是必须、工程实践证的是可行——三者不可互相替代。
- 只有单链支撑的主张,标「单证据,弱」,不得作为方案主线依据。
- 多路并行调研(方法论/开源生态/竞品各一路)时,各路独立完成后先交叉验证再合成,单路结论不直接进报告。

## 亲测优先级

- 子 agent 之间结论矛盾(一个说链接可用、一个说已下线)→ **亲测定夺**,把亲测结果写成诚实表述(如「落地页可打开,但完整流程已下线」)。
- 案例清单最终分两档交付:「可实测清单」(链接亲测可走通)与「仅借鉴形式」(已下线,标『勿引链接』)。
