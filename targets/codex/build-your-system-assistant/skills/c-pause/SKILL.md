# Codex 适配说明

- 这是原 Claude assistant 命令 `c-pause` 的 Codex skill 版本。
- 文中旧的斜杠命令现在都表示对应 skill，而不是可直接输入的 slash command。
- 如果需要用户做选择，优先使用当前环境支持的选项式提问；若不支持，就给出简短可直接选择的选项。
- 遇到 Claude 专属语法时，使用 Codex 等价方式完成，不要照搬旧工具名。
- 在 Vault 场景下默认当前工作目录就是 Vault 根目录；若不是，先确认路径。

在任务转换点记录间隙日志（Interstitial Journaling）。

**用户提供的参数**

## Interstitial Journaling 核心理念

在任务切换的"间隙"进行 30 秒的心理重置：
- 清空上一任务的心理残留
- 让拖延行为可见（写下"又刷了 20 分钟"比默默刷更有威慑力）
- 为下一任务腾出认知空间

## 执行流程

### 0. 检查初始化状态

先检查 `60-Memory/profile.md` 是否存在。如果不存在，提示用户先运行 ``a-setup`` 完成初始化。

### 1. 获取当前时间

读取当前时间（GMT+8），格式化为 `HH:MM`。

### 2. 智能检测上下文（可选）

如果用户没有提供 用户提供的参数：
- 尝试读取最近的 Codex 活动作为"刚完成"的默认值
- 如果无法检测，直接询问用户

### 3. 收集间隙信息

使用 当前环境的提问能力收集以下信息：

**问题 1**: 刚完成什么？
- 如果有 用户提供的参数 或检测到的活动，作为默认选项
- 用户可以确认或修改

**问题 2**: 感受如何？
- 😊 顺利 - 比预期好或按计划完成
- 😐 一般 - 正常推进，无特别感受
- 😔 卡住 - 遇到困难或阻塞
- 🎯 专注 - 进入心流状态
- 📱 分心 - 被打断或自己跑偏了

**问题 3**: 下一步是什么？
- 自由输入

**问题 4**: 备注（可选）
- 任何想记录的额外想法

### 4. 写入间隙日志

将记录追加到 `00-Inbox/{YYYY-MM-DD}.md` 的 `## 间隙日志` 区块。

如果该区块不存在，在文件开头创建：

```markdown
## 间隙日志

### {HH:MM}
- **完成**: {刚完成的内容}
- **感受**: {emoji} {感受描述}
- **下一步**: {下一步内容}
- **备注**: {备注内容，如果有}

---
```

如果区块已存在，在区块内追加新记录（最新在最上面）。

### 5. 反馈用户

```
⏱️ {HH:MM} | 间隙记录已保存
感受: {emoji}
下一步: {下一步内容}
```

## 快速模式

如果用户提供了完整的 用户提供的参数 且格式清晰，可以跳过交互：

**触发条件**：内容包含以下任一模式
- `刚完成 xxx，接下来 yyy`
- `完成了 xxx，下一步 yyy`
- `xxx 做完了，接下来 yyy`

**快速模式流程**：
1. 自动解析"完成"和"下一步"
2. 感受默认为 😐 一般（除非内容包含情绪关键词）
3. 直接写入，不询问确认

**情绪关键词识别**：
- 😊 顺利：顺利、不错、搞定、完美、提前
- 😔 卡住：卡住、困难、问题、bug、搞不定
- 🎯 专注：专注、心流、沉浸、高效
- 📱 分心：分心、刷、跑偏、浪费、又

## 示例

### 交互模式

```
用户: `c-pause`

Claude:
⏱️ 14:32 | 间隙记录

[选项式提问: 收集间隙信息]

用户回答后:
⏱️ 14:32 | 间隙记录已保存
感受: 😊 顺利
下一步: 开会准备
```

### 快速模式

```
用户: `c-pause` 刚完成文章初稿，感觉不错，接下来准备开会

Claude:
⏱️ 14:32 | 间隙记录已保存
感受: 😊 顺利
下一步: 准备开会
```

### 分心记录

```
用户: `c-pause` 又刷了20分钟Twitter，该回去工作了

Claude:
⏱️ 10:46 | 间隙记录已保存
感受: 📱 分心
下一步: 回去工作
```

## 重要规则

1. **时间戳必须准确** - 使用 GMT+8 当前时间
2. **30秒内完成** - 间隙记录要快，不能变成负担
3. **感受真实记录** - 分心就是分心，不美化
4. **下一步要具体** - 不是"继续工作"，而是"完成报告第二段"
5. **当前目录就是 Vault** - 使用相对路径
