# Data Editorial

```yaml
id: data-editorial
use-for: rankings, trends, metrics, comparisons, chart-led article images
background: clean off-white #F7F5EF or solid deep navy selected once per set
palette: high-contrast neutral data marks with one semantic accent system
accent: one color used consistently for the highlighted series or category
medium: precise editorial data graphic with restrained material depth
lighting: even and quiet so shadows never alter a mark or axis
composition: data is dominant, ordered reading path, secondary scene elements stay outside the plot
text-rules: exact categories, values, units, order, and one nonduplicated labeling method
avoid: duplicate category labels, distorted axes, occluded marks, unlabeled decoration, fake interface chrome, watermark, multiple focal points
```

Material depth may separate layers but may not change apparent values, baselines, bar lengths, curve positions, or category order.

Use exactly one label per category. When the required string already combines category, value, and unit, render that complete string once beside its mark. Never split a required combined label into a standalone category name plus a repeated value label. Do not repeat category names on an axis, legend, row header, or annotation when the complete label already appears.
