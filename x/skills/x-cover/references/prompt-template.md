# 封面 prompt 金模板

> 纪律真源 = `../scripts/cover-gen.sh` gen 模式的注入段。A 段给一键流程用(只写画面);B 段给桌面 app 手动兜底用(纪律全文照抄,别改写)。

## A. 画面内容骨架(cover-gen.sh gen 用,纪律句会被脚本注入)

```
画面(〖皮肤:深色终端风 / 暗色科技风 / 用户指定〗,按 2.5:1 横幅构图):
- 背景:〖如 深 navy 渐变 #13203a → #0b1322〗。
- 主标题:「〖≤7 字大字钩子〗」(粗体白字);下方副标「〖375px 可读的一句〗」(〖强调色〗)。
- 〖单一图形钩子 = 文章核心观察的视觉化,全图唯一焦点〗
- 〖版式结构:哪几行/哪几列/焦点在谁 —— 从下方 8 类里选〗
- 主数字:〖如 48,000+ stars,必须带单位;没有就删本行〗。
- 配色锁 hex:〖≤2 主色 + 1 强调色,逐个写 hex〗。
```

## B. 桌面 app 手动兜底(A 的画面内容 + 下面整段一起贴)

```
用 imagegen skill 的 image_gen 工具直出整张 X 文章封面(含全部文字),size 参数传 "2400x960"(= 2.5:1,X 封面框)。若工具对尺寸取整,不必重试凑精确像素——成图宽高比在 2.45:1 到 2.55:1 之间即合格。整张一次性直出,按 2.5:1 横幅构图。

硬纪律(违反任何一条 = 重做):
1. 禁止事后本地叠字/排版合成/二次编辑;禁止写任何渲染脚本(Swift/PIL/ImageMagick/SVG 一律禁止)。所有文字由 image_gen 生成时直接画进图里。
2. 硬禁 glow/外发光/渐变雾/霓虹光晕;文字线条锐利实心。
3. 大数字必须带单位;趋势只画单条干净曲线,不画散点+阶梯柱拼凑的伪图表。
4. 中文逐字正确(Hiragino Sans GB 观感)、英文/仓库名逐字符正确;图标/方框/箭头/星形自己画,不用 emoji。
5. 首张有硬伤(乱码/伪图表/glow/比例错)最多自查重出 1 次;微调级瑕疵不重出,如实报告。
```

出图后:`bash "${CLAUDE_PLUGIN_ROOT}/skills/x-cover/scripts/cover-gen.sh" from <原图> <文章目录>`

## 版式 8 类速查(信息怎么摆,数据类文章先定版式)

| 类 | 何时用 | 代表版式 |
|---|---|---|
| 排行榜 | 完整榜单、多条目对比 | 竖向 Top-N / 领奖台 / Tier 分层 / 横向条形 / 榜首聚光 |
| 大数字 | 一个数字就是全部故事 | Hero 巨数 / 数字+涨跌 / 里程表 / 数字+sparkline |
| 趋势 | 增长/走势 | 单条折线面积 / 节点标注增长 / 沉寂→爆发 / 斜率逆袭 |
| 对比 | 两者对决、前后对照 | VS 分屏 / 对比表(主角列高亮)/ 前后卡 / 占比条 |
| 数据网格 | 多指标一屏看全 | KPI 网格 / Bento / 仪表盘 / 三栏指标 |
| 结构关系 | 历程/机制/格局 | 时间线 / 流程管线 / 四象限 / 层级树 / 分布 |
| 标注 | 拆解一个对象 | 产品卡引线 / 界面圈点 / 带标签示意图 |
| 清单 | 要点/理由/卖点 | 编号清单 / 图标网格 / 要点卡 |

**版式硬规则**:① 单一焦点 = 信息钩子,次要信息退层级;② 2 秒内 get"这是什么/主角是谁/关键数字";③ 375px 缩略图主标题/核心数字必可读;④ 数字右对齐、基线一致。
